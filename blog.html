Get Your AWS Marketplace SaaS Registration Page Live in Minutes

by Kevin Kennedy | on 13 FEB 2026 | in AWS Marketplace, AWS CloudFormation | Permalink · Comments · Share

Listing your SaaS product on AWS Marketplace puts it in front of millions of AWS customers who can subscribe and pay through their existing AWS account. Whether you use contract pricing, usage-based subscriptions, or a hybrid model, one thing every SaaS listing needs is a registration landing page connected to the AWS Marketplace APIs.

Every SaaS product listed on AWS Marketplace needs a registration page. When a customer subscribes to your product, AWS Marketplace redirects them to your page so they can provide their contact details and get started, or register directly into your SaaS application. You need to host this page, integrate it with the AWS Marketplace APIs (such as ResolveCustomer and BatchMeterUsage), and store the registration data.

Setting this up typically involves deploying multiple AWS services, writing Lambda functions, editing HTML, and wiring everything together. It works, but it takes time, especially if you're doing it for the first time.

In this post, we walk through a sample solution that does all of this for you. A single AWS CloudFormation template deploys everything, including the branded registration page itself. You provide your company name, branding preferences, and email as CloudFormation parameters, and the stack takes care of the rest.

What you get

One CloudFormation stack. One deployment. Here's what it creates:

  • A branded registration page hosted on Amazon CloudFront with your logo, colors, and welcome message. Deployed automatically, no file uploads needed
  • A registration API that validates the customer's token with AWS Marketplace and stores their details in Amazon DynamoDB
  • Email notifications via Amazon SNS every time a new customer registers
  • A metering pipeline for usage-based pricing. Collects, aggregates, and submits usage data to AWS Marketplace hourly
  • Subscription lifecycle handling that automatically updates customer status when subscriptions are created, amended, or cancelled
  • Optional custom domain support with automated SSL certificate setup
Figure 1: What gets deployed — CloudFront serves your branded page, API Gateway handles registrations, DynamoDB stores subscribers, and metering runs automatically in the background.

How it works for your customers

From your customer's perspective, the experience is straightforward:

  1. They find your product on AWS Marketplace and subscribe.
  2. AWS Marketplace redirects them to your registration page.
  3. They see a branded page with your company name and logo.
  4. They fill in their company name, contact person, phone, and email.
  5. They click "Complete registration" and see a confirmation message.

Behind the scenes, the system validates their subscription with AWS Marketplace, stores their details, and sends you an email notification — all in a few seconds.

One page for all your products

Key point: You only need to deploy this once. The same registration page works for all your SaaS products — current and future.

When AWS Marketplace redirects a customer, it includes a token that identifies which product they subscribed to. The registration system reads the product code from that token automatically.

To add a new product, just set its fulfillment URL in the AWS Marketplace Management Portal to the same CloudFront URL.

Getting started

Prerequisites

  • An AWS account with Marketplace seller registration
  • A SaaS product listing (Limited or Public)
  • AWS CLI configured, or access to the AWS Console

Step 1: Deploy the template

Download the CloudFormation template from the GitHub repository and deploy it in us-east-1. You can use the AWS Console or the CLI:

aws cloudformation create-stack \
    --stack-name my-marketplace-landing \
    --template-body file://cloudformation-template.yaml \
    --capabilities CAPABILITY_IAM \
    --parameters \
        ParameterKey=CompanyName,ParameterValue="Your Company" \
        ParameterKey=AdminEmail,ParameterValue=admin@example.com \
    --region us-east-1

The only required parameters are CompanyName and AdminEmail. Branding and custom domain parameters are optional. Deployment takes about 5 minutes.

Step 2: Confirm the SNS subscription

Check the admin email inbox and confirm the SNS notification subscription. This enables email alerts for new registrations and subscription changes.

Step 3: Set the fulfillment URL

Get the CloudFront URL from the stack outputs:

aws cloudformation describe-stacks \
    --stack-name my-marketplace-landing \
    --query 'Stacks[0].Outputs' \
    --region us-east-1

Go to the Marketplace Management Portal and set the CloudFront URL as the fulfillment URL for your product.

Figure 2: CloudFormation stack outputs with the CloudFront URL to use as your fulfillment URL

That's the entire setup. Your registration page is live.

Branding your page

The template accepts four optional branding parameters:

ParameterWhat it doesDefault
LogoUrlDisplayed in the header barCompany name text only
PrimaryColorButtons, focus states, accents#0073bb (AWS blue)
HeaderColorTop navigation bar background#232f3e (AWS dark)
WelcomeMessageSubtitle below the page heading"Complete your registration to get started"

These are CloudFormation parameters, so you can update them later with a stack update.

Using your own domain

By default, your page is served from a CloudFront URL like d1234abcdef.cloudfront.net. If you want to use your own domain (for example, register.yourcompany.com), the template supports two options:

If your domain is on Amazon Route 53: Provide the CustomDomain and HostedZoneId parameters. The stack creates the SSL certificate, validates it via DNS, and adds the DNS record. Fully automated.

If your domain is on another provider (GoDaddy, Cloudflare, etc.): Create an SSL certificate in AWS Certificate Manager in the us-east-1 region first, then provide the CustomDomain and AcmCertificateArn parameters. After deployment, add a CNAME record pointing your domain to the CloudFront URL.

Metering for usage-based pricing

If your product uses usage-based pricing (SaaS Subscription or Contract with Consumption), the stack includes a complete metering pipeline that runs automatically:

  1. Your application writes usage records to a DynamoDB table (for example: "customer X used 100 API calls on product Y").
  2. Every hour, a background job aggregates the pending records.
  3. The aggregated usage is submitted to the AWS Marketplace BatchMeterUsage API.
  4. Each record is marked as processed, and cumulative totals are tracked per customer.

For contract-only products, the metering resources are still deployed but not required. They sit idle at negligible cost.

For detailed metering integration instructions and code examples in Node.js and Python, see our companion post: Integrating Usage Metering with Your AWS Marketplace SaaS Product.

What happens when subscriptions change

The stack automatically listens for subscription lifecycle events from AWS Marketplace:

EventWhat happens
Customer subscribesRegistration stored, you get an email
Subscription amended (upgrade/downgrade)Status updated, you get an email
Subscription cancelledStatus set to "cancelled", you get an email
License updated or deprovisionedStatus updated accordingly, you get an email

No polling, no manual checks. You always have an up-to-date view of your subscribers in DynamoDB, and you're notified of every change via email.

Things to know

  • Deployment takes about 5 minutes. If you're using Route 53 for a custom domain, certificate validation may add a few more minutes.
  • The fulfillment URL is a manual step. After deployment, you need to set it in the Marketplace Management Portal. This can't be automated because it's part of your product listing configuration.
  • Deleting the stack? Use --retain-resources RedirectFunction in the delete command. The Lambda@Edge function takes up to an hour to fully clean up from CloudFront edge locations. You can delete it manually later.
  • Updated your branding? After a stack update, run a CloudFront cache invalidation (aws cloudfront create-invalidation --distribution-id <ID> --paths "/*") to see changes immediately.
  • All Lambda functions use Node.js 18 with AWS SDK v3, which is included in the runtime — no dependencies to manage.

Cleanup

To remove everything:

aws cloudformation delete-stack \
    --stack-name my-marketplace-landing \
    --retain-resources RedirectFunction \
    --region us-east-1

Conclusion

Getting a SaaS registration page live on AWS Marketplace doesn't need to be a multi-hour project. This tool generates a single CloudFormation template that handles the infrastructure, the registration flow, the branding, metering, and subscription lifecycle — so you can focus on your product instead of plumbing.
